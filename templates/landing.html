Breaking Communication Barriers with SignAI

A final-year engineering evaluation project demonstrating high-performance real-time translation of Indian Sign Language (ISL) gestures to text and speech. Designed with accessibility-first principles.

Launch Translation App

Core Objectives & Design

01

Real-Time Joint Tracking

Leverages MediaPipe's high-fidelity hand landmarker API to track 21 coordinate joints on each hand dynamically in the browser environment.

02

High-Contrast Accessibility

Built using semantic HTML5 layouts, visible focus ring states, aria-live regions, and color contrast ratios exceeding WCAG AA criteria.

03

Tactile Audio Output

Translates gestured letters into words and continuous sentences. Features high-fidelity client-side Web Speech TTS synthesis.

04

Intelligent Activation

Features OpenCV face-detection logic that unlocks and locks the translation pipeline only when a user is actively facing the screen.

System Architecture & Technical Rigor

The system utilizes a modular backend and client architecture. A background thread processes camera frames using OpenCV and extracts MediaPipe landmarks. These landmarks are passed through a pipeline that checks geometrical joint configurations (angles, finger extension state, and relative coordinates) to map hand shapes to ISL representations.

Translation Scope & Limitations

✓ In-Scope Gestures

  • 15 ISL Letters: Static, one-handed signs for A, B, C, D, E, F, I, K, L, O, S, U, V, W, and Y.
  • Digits 0 to 9: Dynamic finger counting. 1–5 are single-handed, 6–9 are two-handed, and 0 is signed as two closed fists.
  • Debounced Confirmation: Gestures must be held for 15 frames to trigger a confirmed character input.

✗ Out-of-Scope (Future Work)

  • Dynamic Letters: Letters requiring motion (e.g. J, Z) are not supported by the static rules engine.
  • Two-Handed Alphabet: Complex alphabetic letters requiring two active hands (e.g. G, H, M, N, P, Q, R, T, X) are out of scope.
  • Sentence Parsing: Natural language grammatical structuring is left as a post-processing step.

Breaking Communication Barriers with SignAI

A final-year engineering evaluation project demonstrating high-performance real-time translation of Indian Sign Language (ISL) gestures to text and speech. Designed with accessibility-first principles and featuring Whole-Word Sign Sequence recognition and split-screen Conversation Mode.

Launch Translation App

Core Objectives & Design

01

Whole-Word Sign Sequence

Goes beyond static poses to track hand landmark coordinate trajectories over a 30-frame sequence buffer, classifying dynamic whole-word ISL signs.

02

Conversation Mode

Designed to run side-by-side with video calls (Zoom/Meet), featuring a two-way transcript and an accessible chat-reply panel for interlocutors.

03

Tactile Audio Output

Translates letters and whole-word signs into words and sentences. Features client-side Web Speech TTS synthesis.

04

Intelligent Activation

Features OpenCV face-detection logic that unlocks and locks the translation pipeline only when a user is actively facing the screen.

System Architecture & Technical Rigor

The system utilizes a modular backend and client architecture. A background thread processes camera frames using OpenCV and extracts MediaPipe landmarks. Static hand configurations are evaluated using geometrical heuristics, while dynamic whole-word signs are parsed as temporal series using Random Forest or LSTM sequence classifiers.

Translation Scope & Limitations

✓ In-Scope Gestures

  • 20 Whole-Word Signs: Dynamic sequence tracking for Hello, Thank you, Please, Sorry, Yes, No, Help, Water, Food, Stop, Wait, Good, Bad, Name, Friend, Family, Doctor, Emergency, Love, and Understand.
  • 15 ISL Letters: Static, one-handed signs for A, B, C, D, E, F, I, K, L, O, S, U, V, W, and Y.
  • Digits 0 to 9: Dynamic finger counting. 1–5 are single-handed, 6–9 are two-handed, and 0 is signed as two closed fists.

✗ Out-of-Scope (Future Work)

  • Complex Sentence Grammars: Real-time mapping from ISL syntactic sequences to spoken English grammatical order.
  • Multi-Signer Generalization: The default sequence models are optimized for single-user environment sessions.
  • Arbitrary Vocabularies: Extensible dictionary mapping beyond the 20 pre-configured whole-word signs.
